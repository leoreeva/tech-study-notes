# Design patterns
Design patterns are general solutions to recurring problems in software design. Think of them as customizable blueprints rather than ready-made code. Unlike algorithms, which define specific steps to achieve a goal, design patterns provide high-level guidance. The same pattern may look different in different programs, as it’s adapted to each context rather than copied directly.

## Classification
The most basic and low-level patterns are often called **idioms**, and they usually apply only to a single programming language. The most universal and high-level patterns are **architectural** patterns: developers can implement these patterns in virtually any language.

In addition, all patterns can be categorized by their purpose:
- **[Creational patterns](creational_patterns.md)** provide object creation mechanisms that increase flexibility and reuse of existing code.
- **[Structural patterns](structural_patterns.md)** patterns explain how to assemble objects and classes into larger structures, while keeping these structures flexible and efficient.
- **[Behavioral patterns](behavioral_patterns.md)** patterns take care of effective communication and the assignment of responsibilities between objects.
