Getting Started with Tor Browser

First Launch

Initial Configuration:

1. Start Tor Browser
2. Connection options:
   ├─ "Connect" - Direct connection (most common)
   ├─ "Configure" - Proxy/VPN settings
   └─ "Censored" - Bridge configuration

Connection Verification:

  • Visit: check.torproject.org
  • Expected result: “Congratulations. This browser is configured to use Tor.”
  • Displays assigned IP address (different from your real IP)

Understanding the Interface

Key Elements

┌─────────────────────────────────────────────┐
│ 🧅 Tor Browser │ Search or enter address  │
├─────────────────────────────────────────────┤
│                                             │
│                                             │
│            Main browsing area               │
│                                             │
│                                             │
├─────────────────────────────────────────────┤
Security Level: Standard | Safer | Safest
└─────────────────────────────────────────────┘

The Onion Menu (🧅):

  • New Identity: Clears all data, creates new Tor circuit
  • New Tor Circuit for this Site: Changes nodes for current site
  • Onion Circuits: View current node chain
  • Security Settings: Adjust protection level
  • Tor Network Settings: Configure bridges/proxies

Security Settings Explained

Level: Standard

✓ All JavaScript enabled
✓ Media auto-plays
✓ NoScript disabled
✓ Best compatibility
Risk: Higher tracking potential

Level: Safer (Recommended)

✗ JavaScript disabled on HTTP sites
✗ Media click-to-play
✓ NoScript enabled (selective)
✓ HTTPS sites mostly functional
Risk: Moderate tracking protection

Level: Safest

✗ JavaScript disabled everywhere
✗ All media click-to-play
✗ Performance optimizations disabled
✗ Many sites may break
Risk: Minimal tracking, maximum security

Changing Security Level:

  1. Click onion icon (🧅)
  2. Select “Security Settings”
  3. Choose desired level
  4. Restart browser if prompted

Anonymous Email Services

ProtonMail

Access: Clear web + Dark web

Features:

  • End-to-end encryption
  • Zero-access encryption (ProtonMail can’t read emails)
  • Swiss privacy laws
  • No logs policy
  • Open source

Account Tiers:

Free:
├─ 500MB storage
├─ 150 messages/day
└─ Limited features

Plus ($5/month):
├─ 5GB storage
├─ 1000 messages/day
├─ Custom domains
└─ Advanced features

Setup via Tor:

  1. Visit protonmail.com through Tor Browser
  2. Click “Sign Up”
  3. Select free plan
  4. Create username and password
  5. Optional: Add recovery email (compromises anonymity)
  6. Complete CAPTCHA
  7. Account ready

TorBox

Access: Tor only (“onion”)

Features:

  • No clear web connection
  • All traffic through Tor
  • Simple interface
  • No frills, basic email

Setup:

1. Visit TorBox "onion" address
2. Select "Sign Up"
3. Enter username
4. Create password
5. Confirm password
6. Complete CAPTCHA
7. Mailbox ready

Technical Details:

  • Server-to-MTA encryption
  • Webmail via SquirrelMail
  • No advanced features
  • Perfect for basic anonymous communication

Bitmessage

Access: Gateway service (requires Bitmessage client)

Features:

  • Decentralized messaging
  • No central servers
  • Proof-of-work anti-spam
  • Encrypted by default

Setup Process:

1. Download Bitmessage client
2. Install on your system
3. Generate address
4. Gateway converts to/from regular email
5. Use through Tor for anonymity

Best For:

  • Advanced users
  • Multiple email addresses
  • High security requirements

Mail2Tor

Access: Tor only

Features:

  • Free anonymous email
  • Webmail interface
  • No registration required
  • Simple and fast

Daily Usage Best Practices

Browsing Habits

DO:

  • Use bookmarks for trusted sites
  • Keep security settings at “Safer” minimum
  • Clear cookies/cache regularly
  • Use unique usernames per site
  • Verify “onion” URLs carefully

DON’T:

  • Maximize browser window
  • Enable JavaScript on unknown sites
  • Download files without scanning
  • Click random links
  • Use real personal information

Communication Security

Email:

  • Use PGP for sensitive communications
  • Separate email per service
  • Never use real name

Chat:

  • Don’t share personal details
  • Verify identity through multiple channels
  • Use private rooms, not public lobbies
  • Don’t trust anyone implicitly

Forums:

  • Create unique persona
  • Separate accounts per forum
  • Don’t reveal identifying information
  • Use VPN in addition to Tor (optional)

Verification Checklist

Before using any service:

  • Research the site/service
  • Check community reviews
  • Verify “onion” URL from multiple sources
  • Test with disposable email first
  • Never deposit large amounts initially
  • Use escrow when available
